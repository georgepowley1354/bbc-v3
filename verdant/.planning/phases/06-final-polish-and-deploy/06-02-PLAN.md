**Interpret results:**
- Parse the JSON output to extract scores for each category

**Common issues to watch for (from research):**
1. **Color contrast** — `text-white/60` and `text-white/40` in Footer.tsx may fail WCAG AA. If flagged, increase opacity: `text-white/60` -> `text-white/70`, `text-white/40` -> `text-white/60`.
2. **Touch target size** — Footer links at `text-sm` with no padding. If flagged, add `py-1` to footer link classes for minimum 44px touch targets.
3. **Image alt text** — Verify all `<Image>` components have non-empty `alt` attributes (already checked in research — should be fine).
4. **Missing heading hierarchy** — Ensure no heading levels are skipped (h1 -> h3 without h2).

**If any category scores below 95:**
- Read the specific audit failures from the JSON report
- Fix each flagged issue in the relevant source file
- Rebuild and redeploy (push to master)
- Re-run Lighthouse to verify the fix

**Cleanup after audit:**
- Delete `lighthouse-report.html` and `lighthouse-report.json` from the project root (do not commit audit artifacts per project conventions in CLAUDE.md)
