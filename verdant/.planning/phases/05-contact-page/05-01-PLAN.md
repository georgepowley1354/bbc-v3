function encode(data: Record<string, string>): string {
  return Object.keys(data)

    .join('&')
}

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → Netlify Forms endpoint | User-supplied form data (name, email, phone, project description) sent via fetch POST |
| Public HTML → Netlify bot | Static mirror form in public/ is world-readable but contains no sensitive data |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Spoofing | ContactForm.tsx fetch POST | mitigate | Netlify honeypot field (bot-field) present in both static mirror and Client Component form — bots that auto-fill all fields are rejected by Netlify |
| T-05-02 | Tampering | ContactForm.tsx client-side validation | accept | Form validation is UX-only; Netlify Forms is the authority. No server-side business logic depends on form correctness. Data is inquiry, not transaction. |
| T-05-03 | Information Disclosure | public/netlify-form-detect.html | accept | File is intentionally public. Contains only field names and option values — no secrets, no PII, no API keys. |
| T-05-04 | Denial of Service | Netlify Forms spam | mitigate | netlify-honeypot="bot-field" on the registered form reduces automated spam. Netlify's built-in spam filtering applies at the platform level. |
| T-05-05 | Elevation of Privilege | form-name body field | mitigate | form-name value is hardcoded to 'contact' in the component. Netlify matches submissions to the registered form by this field — cannot escalate to other forms since only one form is registered. |
</threat_model>

## What NOT to Do

**DO NOT add 'use client' to page.tsx.** The page is a Server Component. ContactForm.tsx is the Client boundary. Marking page.tsx as a Client Component breaks static generation for this route.

**DO NOT skip public/netlify-form-detect.html.** Without this file, Netlify's bot will never see the form during build time. The Client Component form is invisible to the bot — it is rendered after hydration. Skipping the static mirror means all submissions return 404 or are silently dropped with no error.

**DO NOT use a standard form POST with action="/.netlify/forms/contact" and page redirect.** The site uses Client Component state for success/error UX. The fetch + encode approach keeps the user on the page and enables the luxury confirmation panel.

**DO NOT forget the hidden form-name input inside the Client Component form element.** Netlify matches fetch submissions to registered forms by the `form-name` field in the POST body. Without it, submissions are discarded silently.

**DO NOT use @tailwindcss/forms plugin classes.** The plugin is not installed (tailwind.config.ts has `plugins: []`). Use raw Tailwind utilities — `appearance-none`, `bg-transparent`, `border-b`, `focus:border-sage`.

**DO NOT remove or modify useReducedMotion() guards on MotionDiv elements.** Every animated Client Component in this codebase passes `undefined` for variants when `prefersReducedMotion` is true. Breaking this pattern fails WCAG 2.1 AA (Success Criterion 2.3.3) and inconsistency with the rest of the codebase will flag in Phase 6 Lighthouse audit.

**DO NOT add page-level entrance animation in page.tsx.** src/app/template.tsx already wraps every page in a fade-in transition. Adding another entrance animation at the `<main>` level causes double-animation.

**DO NOT modify Navigation.tsx or Footer.tsx.** Research verified both files already include 'Contact' in their navLinks arrays and both "Start a Project" CTAs already href="/contact". Any edits here risk breaking the working nav.
</anti_patterns>
