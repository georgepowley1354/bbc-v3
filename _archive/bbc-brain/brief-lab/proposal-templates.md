- [Page 2]
- [Page 3]

- [Feature 1]

| Milestone | Date |
|-----------|------|

| Build complete | [date] |
| Launch | [date] |

## SOW Template

```markdown
# Scope of Work — [Client Name]
Date: [date]
Project: [project name]

---

## 1. Project Overview

[Client name] is a [type of business]. This SOW covers [what we're building] to [the goal — more bookings, cleaner online presence, etc.].

## 2. Deliverables

1. [Deliverable 1 — be specific]
2. [Deliverable 2]
3. [Deliverable 3]
...

## 3. What's Not Included

The following are NOT included in this scope unless agreed via change order:
- Copywriting
- Photography or video production
- Logo design
- Ongoing maintenance (see Tier 3 retainer)
- [Anything specific to this project]

## 4. Timeline

| Milestone | Date |
|-----------|------|
| Discovery / kickoff | [date] |
| Design mockups delivered | [date] |
| Design revision round | [date] |
| Build complete | [date] |
| Client review | [date] |
| Launch | [date] |

## 5. Investment

**Total project fee:** $[amount]

**Payment schedule:**
- 50% deposit: $[amount] — due before work begins
- 50% balance: $[amount] — due before launch

## 6. Terms

**Revisions:** This project includes [2] rounds of design revisions and [1] round of copy/content revisions after build. Additional rounds billed at $[rate]/hr.

**Approvals:** Client must approve design before build begins. Scope changes after approval require a change order.

**Change orders:** Any additions outside this scope require written agreement and may affect timeline and cost.

**Kill fee:** If the project is cancelled after work begins, the deposit is non-refundable. If cancelled after the build phase begins, 75% of the total fee is due.

**Launch:** Client is responsible for providing access to domain registrar for DNS changes at launch.

---

Agreed and accepted:

Client signature: _________________________ Date: _________

Jorge / BBC: _____________________________ Date: _________
```

## Proposal Hook Types

Use one of these openings for proposals — adapted to the client's situation:

| Hook | When to Use | Example |
|------|------------|---------|
| **1 — Problem-First** | Client described a clear pain | "Your current site makes it hard for customers to find your hours, menu, or contact you — and that's costing you bookings." |
| **2 — Results-Lead** | You have a similar reference project | "We built a site for [similar client] that increased their reservation requests by [X]. Here's how we'd do the same for you." |
| **3 — Comparison** | Client is evaluating options | "Most agencies will send you a 20-page proposal. This one fits on two pages because the scope is clear." |
| **4 — Social Proof** | Referral-based lead | "You came to us through [referrer] — we built their site and the approach here would be similar." |
| **5 — Investment Frame** | Budget-sensitive client | "A custom site at this scope is a one-time investment. Here's exactly what you're getting and why the number makes sense." |
