# Brief Lab — Workspace Context

## What This Is

Where client briefs, proposals, and scopes of work are written and refined. Every client engagement starts here.

---

## Files in This Workspace

| File | What It Contains |
|------|----------------|
| `brief-framework.md` | Discovery → brief → SOW process with question sets |
| `proposal-templates.md` | Templates for proposals, SOWs, and change orders |
| `discovery/` | Client discovery notes (one file per client) |
| `proposals/` | Draft and final proposals |
| `scopes/` | Signed scopes of work |

---

## What to Load by Task

### Writing a new proposal
1. `agency-identity/voice-and-tone.md` → "Agency Voice Rules" + "Pitch Tone"
2. `agency-identity/who-jorge-is.md` → "One-Sentence Version" + "Credibility Rule"
3. `brief-framework.md` → full file
4. `proposal-templates.md` → relevant template section

### Writing a scope of work
1. `brief-framework.md` → "SOW Structure" section
2. `proposal-templates.md` → "SOW Template" section
3. Relevant `discovery/[client]-discovery.md` → full file

### Running a discovery session
1. `brief-framework.md` → "Discovery Question Set" section

### Review an existing proposal
1. `proposal-templates.md` → "Proposal Review Checklist"

---

## Process: From Lead to Signed Scope

```
1. Lead comes in → capture in project-engine/lead-capture.md
2. Discovery call → notes in discovery/[client]-discovery.md
3. Proposal written using brief-framework + templates → proposals/[client]-proposal-draft.md
4. Client reviews, feedback round
5. Final proposal → proposals/[client]-proposal-final.md
6. SOW written and signed → scopes/[client]-sow-signed.md
7. Brief prepared for build → copy to build-studio/workflows/01-briefs/[client]-brief.md
```
