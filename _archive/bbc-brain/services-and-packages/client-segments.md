# Client Segments

## Client Types BBC Works With

### Ideal Clients

**Local service businesses**
- Restaurants, cafes, bars, food trucks
- Contractors, landscapers, tradespeople
- Gyms, studios, personal trainers
- Local consultants and professional services

**Creative businesses**
- Artists, photographers, musicians
- Independent retailers
- Event spaces and planners

**Common traits:** Real business, real revenue, clear sense of what they do, willing to invest appropriately, responsive communicators.

---

## Fit Scoring

Use this to quickly assess a new lead. If they score 7+, pursue. If 4-6, proceed carefully. If under 4, pass.

| Signal | Score |
|--------|-------|
| Has a clear, specific ask | +2 |
| Has an existing business (not pre-launch idea) | +2 |
| Has seen BBC's work and referenced it | +2 |
| Was referred by a current or past client | +2 |
| Has a real timeline with a reason | +1 |
| Has content ready (or knows they need to get it) | +1 |
| Budget signal is in range | +2 |
| **Red flags (subtract)** | |
| Can't describe what they do in one sentence | -2 |
| "Just want something simple" with complex needs | -2 |
| Has worked with 3+ agencies and is unhappy with all | -2 |
| No budget signal at all | -1 |
| Wants to "see what you come up with" | -2 |
| Needs it done in 2 weeks | -1 |

## Red Flags

These patterns predict difficult projects. Note them in discovery notes. If multiple are present, raise the price or pass.

- **Vague brief:** "I want a website that really pops." Can't describe deliverables. Every round of revisions will be subjective.
- **Serial agency hopper:** Was burned by last 2 agencies. Often the client is the problem.
- **Designer client:** Has strong opinions about every pixel, doesn't trust the process.
- **Free feature creep:** Starts with 4 pages, ends up asking for a booking system, inventory management, and a member portal.
- **No deposit resistance:** "Can we start now and settle up later?" — cash flow problem or commitment problem.
- **Silent client:** Goes weeks without responding, then needs everything in 48 hours.

---

## How to Handle Borderline Fits

If the client is borderline but has potential:
1. Scope tightly — explicit "not included" list in the SOW
2. Charge the higher end of the range
3. Add a change order clause that's easy to invoke
4. Set communication expectations upfront (response time, approval windows)
